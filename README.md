<h1>Каталог фильмов</h1>
<h2>Описание проекта</h2>
Это консольное приложение для управления личным каталогом фильмов. Программа позволяет пользователям:

✔ Просматривать список всех фильмов с указанием года выпуска и статуса просмотра;

✔ Добавлять новые фильмы в каталог с указанием названия и года;

✔ Отмечать фильмы как просмотренные;

✔ Искать фильмы по году выпуска.

Данные фильмов хранятся в локальном JSON-файле (movies.json), что обеспечивает простое и удобное сохранение информации между запусками приложения. Приложение ориентировано на практичность и минимализм, используя текстовый интерфейс для максимально легкого взаимодействия.

<h2>Основные возможности</h2>
✔ Загрузка и сохранение каталога из JSON-файла

✔ Добавление фильмов с автоматическим назначением уникального ID

✔ Отображение списка всех фильмов с отметкой «Просмотрен» или «Не просмотрен»

✔ Поиск фильмов по году

✔ Отметка фильма как просмотренного по названию

Технические требования
Python версии 3.13 и выше

Рекомендуется использовать виртуальное окружение для установки зависимостей

<h2>Установка и подготовка</h2>
Скачайте или клонируйте репозиторий проекта:

<pre>
git clone https://github.com/Bibindum1/Movie.git
cd Movie
</pre>

<h2>Создайте и активируйте виртуальное окружение:</h2>

<pre>
python -m venv venv
</pre>
<pre>
venv\Scripts\activate
</pre>

<h2>Установите необходимые зависимости (для тестирования):</h2>

<pre>
pip install pytest
</pre>
Убедитесь, что рядом с вашим скриптом находится файл <pre>movies.json</pre>. Если его нет, программа создаст его автоматически при первом запуске.

<h2>Структура проекта</h2>
<pre>
main.py — главный скрипт с циклом меню и пользовательским интерфейсом.
</pre>
  <pre>
logic.py — модуль с функциями для работы с фильмами: загрузка, сохранение, добавление, поиск, отметка просмотренных.
</pre>
  <pre>
movies.json — файл данных, содержащий список фильмов.
</pre>
  <pre>
test_logic.py — тесты, написанные с использованием pytest для проверки логики проекта.
</pre>

<h2>Запуск программы</h2>
Для запуска приложения выполните следующую команду в терминале:

<pre>
python main.py
</pre>
После запуска вы увидите меню с доступными действиями. Просто введите номер пункта и следуйте подсказкам:
<pre>
Каталог фильмов
1 - Показать все фильмы
2 - Добавить фильм
3 - Отметить фильм как просмотренный
4 - Найти фильмы по году
0 - Выход
</pre>
<h2>Описание функций меню</h2>

1 — Показать все фильмы: Отобразит полный список фильмов с их ID, названием, годом выпуска и статусом просмотра.

2 — Добавить фильм: Позволит ввести название и год нового фильма, который будет добавлен в каталог. ID будет сгенерирован автоматически.

3 — Отметить фильм как просмотренный: Запросит название фильма и пометит его как просмотренный, если такой фильм есть в списке.

4 — Найти фильмы по году: Запросит год и покажет все фильмы, выпущенные в этот год.

0 — Выход: Завершение работы программы.

<h2>Формат данных</h2>
Каждый фильм в JSON-файле хранится как объект с полями:

<pre>
{
  "id": 1,
  "title": "Название фильма",
  "year": 2020,
  "watched": false
}
</pre>
<pre>
id — уникальный идентификатор (целое число)

title — название фильма (строка)

year — год выпуска (целое число)

watched — статус просмотра (логическое значение)
</pre>

<h2>Запуск тестов</h2>
В проекте используется фреймворк pytest для модульного тестирования функций логики.

<h2>Установка pytest</h2>
Если pytest еще не установлен, выполните:

<pre>
pip install pytest
</pre>

<h2>Запуск тестов</h2>
В корне проекта выполните команду:

<pre>
pytest
</pre>
Pytest автоматически найдет все тестовые функции, расположенные в файлах test_*.py или *_test.py, и выведет результаты тестов с подсветкой успехов и ошибок.

<h2>Что тестируют тесты</h2>
Корректную загрузку списка фильмов из файла, в том числе с пустого или поврежденного JSON.
<pre>
✔ Сохранение фильмов в JSON файл.

✔ Добавление нового фильма с правильным подсчетом ID.

✔ Отметку фильма как просмотренного.

✔ Поиск фильмов по году.
</pre>
Это поможет гарантировать стабильную работу проекта и быстро выявлять возможные ошибки при изменениях.
